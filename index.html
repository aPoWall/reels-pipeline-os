как работает – pipeline за 30 секунд

короткий рилз собирает агент из одной записи: транскрипт даёт word-таймкоды → агент режет на смысловые блоки → решения пишутся в EDL (единый источник истины) → overlay-дорожки кладутся независимо → дешёвый draft на проверку → QA → финал 1080×1920. рецепт хранится как данные: стиль можно закрепить, повторить и форкнуть.

запись→ transcript→ source blocks→ EDL · SSOT→ overlay lanes→ draft→ QA→ final 1080×1920

блок-эдитор – как это водится руками

блок-эдитор (skill reel-block-edit) – single-file HTML поверх стека. хороший дубль становится переиспользуемым cut map + overlay-дорожками. собираешь рилз визуально, экспортируешь EDL+manifest, рендерер собирает финал.

source blocks
фразы из транскрипта тегируются по роли (hook/tension/turn/…); каждый блок самодостаточен и переставляем
branch surface
одна cut-карта → много веток (clean/captions/micro/low-rail/soft-card); baseline никогда не перезаписывается
trim
точная подрезка блока слева/справа по source-диапазону, live-preview из исходника
overlay lanes
caption/card/rail/stamp как независимые дорожки поверх cut map
export
manifest realself.timeline.json + EDL → рендерер (moviepy/ffmpeg сейчас, Remotion/OTIO дальше)

пять короткоформатных приёмов – детали поверх linear long-chain

1

hook-бюджет

первые 3 секунды решают всё

длинный формат может разогреваться – у короткого ролика бюджета нет: pattern-interrupt + текстовый хук + curiosity gap в кадре сразу, иначе скип до досмотра.

алгоритм 2026: 3s-retention – главный фильтр до того, как ролик вообще получит охват.

mode · любой · hook=auto
2

микро-ритм

0.08–0.18s резы по битам речи

длинный темп линейной цепочки в reels мёртв. короткий живёт на частых hard/soft-резах синхронно с речью + punch-zoom 2–4 раза на ключевых словах.

cut points строго в паузах транскрипта (word-times), никогда на слове – иначе рвань.

build_crosscut · beat-synced
3

sound-off дефолт

35% смотрят без звука

kinetic captions обязательны: viewer-facing, нижняя зона, max 2 строки, подсветка только хук-слов.

текст добавляет смысл зрителю, служебные labels остаются вне кадра.

mode · caption_only · no_limits
4

один аудио-спайн

A-B чередование аудио = смерть рилза

главная причина плохих reels – склейка речи из разных дублей/источников. один непрерывный аудио, картинка меняется через visual_swap не трогая звук.

при нескольких дублях выбираем лучший по речи, не по картинке – речь не починить, визуал улучшается grade/оверлеями.

audio-first · lipsync_render
5

trial-петля

тест до полного релиза

2 версии монтажа → trial на не-подписчиках 24–48ч → смотрим watch-time, не views → залетел → share to followers + обычный пост в тему (momentum).

короткий формат позволяет дешёвый A/B одной переменной (хук/длина/стиль).

Trial Reels · ≤5/день

итог короткого формата: long-form discipline даёт план и draft-петлю, portable kit даёт preflight и handoff, но короткую механику – hook-бюджет, микро-ритм, sound-off, аудио-спайн, trial-A/B – несёт наш reels-слой.

как это собрано – коллаборация двух агентов

этот рецепт собрали два агента в связке, не один. Codex-сессия держит приватный editing-state (реальный EDL / проект / рендеры), а дашборд-агент (Claude Code) публикует только generic-механику. они общаются напрямую по murmur (agent-to-agent): Codex режет и собирает ветки → отдаёт generic-дельту (слои, EDL-схема, контракты) → дашборд-агент санитизирует и кладёт в публичный стек. так личный контент остаётся приватным, а в open-репо едет только воспроизводимая структура. границу владения и правило no-personal-data держат обе стороны (skill stack-compare + CONTRIBUTING).

шесть заимствований – что втягиваем в Pipeline OS

механики из long-form discipline, portable kit и open-source video agents. ранжировано по соотношению эффект/усилие. верх ряда = самый высокий рычаг.

1

наибольший рычаг

cut-plan → OpenTimelineIO

монтажный план как единый источник правды совпадает с отраслевым стандартом OTIO: JSON-граф клипов, RationalTime, transitions/markers, адаптеры в FCP/Resolve/Premiere.

наш realself.timeline.v1 уже близок (tracks/items, sourceStart/End). переход на .otio даёт frame-accuracy + round-trip с любым NLE + готовые рендереры.

apply → realself.timeline.v1 → .otio схема
2

наибольший рычаг

двухъярус draft→final

180p ultrafast черновик из того же плана → проверка нарезки/таймингов за секунды → 1080p hero только после approve. версии v01.. вместо перезаписи.

сейчас мы рендерим много полных вариантов (дорого). draft-петля = быстрее итерации, дешевле compute, история итераций видна.

apply → render_queue: draft-tier перед hero
3

высокий рычаг

ElevenLabs Scribe для word-times

Scribe v2 = 3.1% WER (2-е место в бенче 2026); Scribe v1-experimental бьёт локальный Whisper на русском вдвое (11.5% vs 23%). word-таймкоды – фундамент точности реза.

наш Whisper medium ~17–20% WER на RU. неточный таймкод = срезанные слова на стыках. Scribe в облаке + Whisper офлайн-fallback.

apply → transcript layer, RU-приоритет
4

средний рычаг

cv2 покадровый зум

push-in считается покадрово в OpenCV – контроль точки приближения, нет дрейфа звук↔картинка (нативные ffmpeg-зумы плывут на длинных).

наш punch-zoom в moviepy ок для коротких, но cv2-kernel детерминированнее. чистая замена эффекта.

apply → dynamics, замена punch_zoom
5

средний рычаг

ProRes 4444 alpha оверлеи

карточки рисуются Pillow и композитятся как ProRes 4444 .mov с альфой – чистая прозрачность без артефактов краёв.

мы кладём PIL-кадр поверх кадра. alpha-дорожка даёт чище края и переиспользуемый оверлей-asset.

apply → overlay lane, .mov alpha export
6

низкий рычаг · аккуратность

детерминированные метаданные

заголовок/описание/таймкоды генерятся из данных эпизода без LLM-вызова – дешевле и точнее соответствует видео.

для youtube-переката нашего контента это уберёт галлюцинации в метаданных; для reels-подписей оставляем человеческий голос.

apply → publish meta (youtube only)

три горизонта развития

research-backed. что строим первым, что – потом.

  1. 1–2 дня · высокий рычаггоризонт 1 – дисциплина плана

    realself.timeline.v1 → OpenTimelineIO; внедрить draft→final двухъярус (videotoolbox draft / libx264 hero); подключить Scribe-транскрипт. ноль новых зависимостей-фреймворков, чистый выигрыш точности и скорости итераций.

  2. 3–5 дней · средний рычаггоризонт 2 – live-preview в браузере

    встроить WebCodecs + Mediabunny в блок-редактор: реальный scrub по source-диапазонам без сервер-рендера. редактор перестаёт быть галереей mp4, становится настоящим NLE. ProRes-alpha + cv2-зум как новые дорожки/эффекты.

  3. 1–2 недели · стратегическийгоризонт 3 – agent × cloud render

    render-адаптер в облако: Remotion Lambda (наш manifest = его inputProps) или Hyperframes (overlays уже html, без React-переписи). изучить X-Cut как референс agent×timeline×Remotion. цель – рилз из контекста одним проходом агента, рендер вне локальной машины.

матрица технологий – fit для нашего стека

★★★ = взять в ядро · ★★ = взять в слой/опцию · ★ = референс/инспирация. чёрные карточки = прямые кандидаты в стек.

OpenTimelineIO (OTIO)★★★
interchange / cut-plan стандарт · ASWF

«современный EDL»: JSON-граф клипов, RationalTime, tracks/clips/gaps/transitions/markers, медиа по ссылке. адаптеры: FCP XML, AAF, CMX3600 EDL, Resolve.

fit: формат для нашего монтажного плана. realself.timeline.v1 → .otio = frame-accuracy + round-trip с любым NLE.
github.com/AcademySoftwareFoundation/OpenTimelineIO
Remotion★★★
programmatic video · react · cloud render

видео как react-компоненты. tracks[]→items[], Player inputProps (JSON-serializable), renderMediaOnLambda, Editor Starter, Timeline-компонент. ~60k weekly downloads.

fit: наш manifest уже зеркалит его модель. облачный рендер-адаптер при масштабе. лицензия Business Source (бесплатно <$1M).
remotion.dev/docs/building-a-timeline
Hyperframes (HeyGen)★★★
html→video · agent-first · open source

«write html, render video, built for agents». композиции = html с data-атрибутами (без react), детерминированный рендер, seekable Puppeteer+FFmpeg capture, Frame Adapter (GSAP/Lottie/CSS), 50+ social-блоков, skills.

fit: самый естественный адаптер – наши оверлеи уже html/PIL. ноль React-переписи. agent-CLI non-interactive = под наш harness.
github.com/heygen-com/hyperframes
X-Cut (MeiGen-AI)★★★
video editing agent · референс-архитектура

опиши идею в чате → агент правит multi-track timeline, рендер в реальном времени на Remotion. skill-архитектура (entry/scenes/operations/tools), «pluggable into Claude». drag-and-drop поверх agent-edit.

fit: ровно наша связка блок-редактор + агент + Remotion, уже собранная. изучить как эталон, не как зависимость.
github.com/MeiGen-AI/X-Cut
OpenScript★★★
mcp video pipeline · edl v2 · rust/typescript

сырой ролик → transcribe → analyze → edit decision list → multi-track timeline → render. MCP tools дают агенту управлять b-roll, voiceover, captions, music, sfx и финальным рендером.

fit: подтверждает направление: EDL+tracks как интерфейс для агента. смотреть как внешний эталон tool surface.
github.com/birchrust/openscript
claude-shorts / OpenCut / ReelForge★★
open-source short-form wave · 2026

общий паттерн: faster-whisper/Whisper word-times, candidate scoring, boundary snapping, silence removal, Remotion render, kinetic captions, 1080×1920 exports.

fit: брать не как зависимость, а как market proof. наш пакет должен быть более переносимым: branch packs + public/private split + config.
claude-shorts · opencut · reelforge
ElevenLabs Scribe v2★★★
speech-to-text · word timestamps

3.1% WER (2-е в бенче Coval 2026), 90+ языков, keyterm prompting (до 1000 терминов), entity detection. v1-experimental: RU 11.5% vs Whisper-medium ~17–20%.

fit: транскрипт-слой, особенно русский контент. word-times = фундамент точности реза. Whisper остаётся офлайн.
stt benchmark 2026
Diffusion Studio Core + Mediabunny★★
browser video engine · webcodecs

browser-движок композитинга на WebCodecs (decode/encode на железе). declarative timeline, captions, silence removal, realtime playback. Mediabunny = «ffmpeg для веба», Remotion спонсирует.

fit: live in-browser preview + клиентский экспорт в блок-редакторе без сервера. редактор → настоящий NLE.
github.com/diffusionstudio/core
WebCodecs + FFmpeg.wasm★★
browser rendering паттерн

разделение: WebCodecs (HW-decode) для scrub/playback в 16мс, FFmpeg.wasm для финального энкода (тоньше bitrate, любые форматы). так устроены все browser-NLE 2026.

fit: live source preview по sourceStart/End в нашем редакторе – то, что SKILL уже хочет, но без сервера.
webcodecs vs ffmpeg.wasm
videotoolbox vs libx264★★
энкодинг · ролевой сплит

HW videotoolbox: быстро (100–1000fps), но артефакты ниже ~5Mbit (нет CRF, только -b:v/-q:v), плохо на градиентах/блюре. libx264 CRF = качество, медленнее.

fit: HW → draft (скорость, артефакты не важны), software CRF → hero. не «или-или», а роли.
ffmpeg videotoolbox
RIFE (frame interpolation)★★
fps · slow-mo · rescue

real-time intermediate flow estimation: ~130fps lite / 125fps full на M1 (7–8мс/кадр 1080p). FILM качественнее на сложном движении, но 5–10× медленнее. SG-RIFE (2026) тянет diffusion-качество.

fit: опц. слой – slow-mo ключевого действия и спасение sub-native fps источника. не для нативной записи.
aifi-benchmark (m1)
Revideo / Motion Canvas★★
code-driven canvas анимация · MIT

generator-based анимация (тgoverning, scene-graph). Motion Canvas – explainer'ы; Revideo – форк с server-render API (renderVideo() из Express/queue/cron). MIT, бесплатно.

fit: для motion-graphic оверлеев и анимированных схем (а-ля shaper-диаграммы в видео), не для полного пайплайна.
github.com/midrender/revideo
Submagic / Opus / Gling / Descript★
saas auto-cut · бенчмарк

edit-by-transcript как индустриальный дефолт: авто-удаление пауз/филлеров, B-roll по транскрипту, magic-zoom, styled captions, virality-score. Descript – «правишь видео как текст».

fit: подтверждение направления + источник UX-идей (zoom-по-транскрипту, B-roll-слайдер). наш ров – стиль/режимы, public/private split и branch packs.
submagic auto edit
FFCreator / editly★
node json-video · легаси-опция

declarative JSON → видео (editly), node-движок коротких роликов (FFCreator). оба MIT, но активность падает (последние коммиты ~год назад).

fit: референс декларативного JSON-видео; для нас перекрыт Remotion/Hyperframes. не брать.
github.com/mifi/editly
