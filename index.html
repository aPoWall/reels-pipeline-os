монтажные протоколы

P0
brief protocol

цель, канал, audience, source type, режим монтажа, privacy boundary and expected artifacts.

P1
source protocol

ffprobe, orientation, fps, audio channels, source map, iPhone HDR/HLG to SDR when needed.

P2
timing protocol

transcript, word timings, pause map, silence/dead-air hints and no cut inside words.

P3
story protocol

source blocks: hook, setup, tension, proof, turn, closer, tail and public-safe summaries.

P4
EDL protocol

`cuts[]` as SSOT with source ranges, output ranges, risk labels and branch status.

P5
overlay protocol

independent lanes: caption, card, rail, proof, stamp, cover. Viewer-facing copy and safe zones.

P6
render protocol

draft, review MP4, text-free base, final 1080x1920, render adapter and deterministic commands.

P7
QA protocol

ffprobe, audio listen, first 3 seconds, safe-zone pass, privacy pass, learning note and publish package.

soft communication layer

hook
why watch

первый смысловой крючок: конфликт, вопрос, обещание пользы or visible proof in the first seconds.

voice
how it speaks

тон, темп, паузы, density captions, persona lane and the amount of visible process.

proof
why believe

экран, объект, timeline, before/after, quote-safe source or action shown inside the frame.

turn
why finish

поворот, вывод, CTA, next branch and learning note for the following edit.

maintenance loop – как личный опыт становится публичным протоколом

01 private pass
render the real reel

сначала делается настоящий монтаж: review MP4, text-free base, cover candidates, audio QA and human notes.

02 survivor
extract reusable mechanics

из удачного прохода забираются только механики: card grammar, caption density, audio policy, cover split and QA checks.

03 skill delta
update private skill

`reel-edit` получает learning note: что закрепить, что запретить, что попробовать на следующем материале.

04 public delta
sanitize for repo

локальные пути, лица, тексты и материалы остаются private. наружу уходит generic branch pack, protocol and source credit.

05 configurator
make it selectable

механика превращается в option: source, goal, mode, route, pack, JSON config and agent prompt.

06 article
explain the method

публичная статья получает короткую формулу: protocol stack, route sessions, source map and private/public boundary.

07 monitor
watch what survives

после публикации собирается не vanity score, а learning signal: first seconds, readability, completion hints and reuse decision.

08 release
ship the public artifact

GitHub Pages держит аккуратную публичную версию: dashboard, STACK.md, PROTOCOLS.md, config prompt and sources tab.

pipeline – как монтировать reels

reels pipeline os держит render-grade spine: source audit, transcript, word-safe cuts, EDL, audio master, overlays, review MP4, text-free base и QA. Любая реализация может заменить renderer, но сохраняет один переносимый editing state.

input
raw `.mov/.mp4`, optional screen proof, creative brief, reference mechanics, publish goal.
analysis
ffprobe source audit, orientation, camera metadata, audio channels, frame scan, source map, transcript with word timings and pause map.
editorial
source blocks, hook/setup/turn/proof/closer roles, one audio spine, word-safe cut points, baseline before experiments.
state
`edit_decision_list.json` as SSOT: `cuts[]`, `overlay_tracks[]`, `render.outputs`, branch status and QA decisions.
surface
caption, card, rail, stamp, proof and cover lanes. copy is viewer-facing, safe-zone aware and public-safe.
outputs
review MP4, final 1080x1920, text-free base, contact sheet, QA report, public recipe without private media.

shaper v2 · 2026-06-15

рабочая последовательность

  1. step 0creative brief / mode selection

    определить цель, нарративную арку, cut policy, overlay density and branch matrix.

  2. step 1deep source analysis

    ffprobe, ориентация, камера, аудио-каналы, frame scan, source map, аудио-источник и visual-only blocks.

  3. step 2transcript + word-level timeline

    слова, паузы, punch words, concept map. cut points ставятся в паузах, не внутри слов.

  4. step 3clean baseline

    silence trim, filler/repeat cleanup, handles 100-260ms, closer tail. baseline сохраняется перед стилем.

  5. step 4branch matrix

    минимум: dry_preserve, caption_only, context branch, optional social_short. Для 10+ вариантов сначала SPECS/render_queue, финалы последовательно.

  6. step 5overlay lanes

    caption, card, rail, stamp, proof. Viewer-facing copy, no internal labels, no source filenames, no `render/QA/branch` на экране.

  7. step 6render + QA

    review MP4, text-free base, contact sheet, ffprobe, volumedetect, first 3 seconds, safe-zone and no-overlap checks.

типовые branch recipes

color return
монохромный или тихий старт → возвращение цвета → restrained captions → clear payoff.
object proof
один физический объект или экранный фрагмент держит доказательство, captions редкие и viewer-facing.
dual voice
coach / critic / meta lanes, no-overlap rules, persona pill в safe-zone, short cuts усиливают voice switch.
collab interruption
interruptions and handoffs как сюжетные beat-ы. turn_map и collabBeats[] обязательны.
cover set
10 cover options: identity references + motif object + clean no-text bases + Pillow caption layer.

libraries stack – из чего собрать pipeline

стек делится на шесть слоёв: core cli, analysis, caption/cover lanes, timeline state, render adapters and QA/monitoring.

core cli

ffmpeg / ffprobe

source audit, crop, scale, concat, audio extraction, AAC, bt709 tags, final diagnostics.

iphone ingest

Apple avconvert

macOS-native path for iPhone HDR/HLG to SDR bt709 before creative grading and render.

speech timing

Whisper / Scribe

word timings, pause map, subtitle base, retranscribe QA after final render.

scene/audio analysis

PySceneDetect / librosa

scene hints, silence/dead-air, rhythm markers, repeated sections and confidence gate before EDL.

proof analysis

EasyOCR / PaddleOCR

screen/system reels: read proof areas, redact private UI and decide whether proof is phone-readable.

caption system

remotion-captioneer

word-level caption JSON, presets, SRT/VTT/ASS exports and reusable caption components for Remotion.

python image

Pillow / numpy

PNG captions, word highlight frames, contact sheets, masks, thumbnails and QA stills.

subtitle lanes

ASS / SRT / WebVTT

portable caption layer for social default, review text and optional burn-in renders.

state model

EDL JSON + OpenTimelineIO

small public-safe EDL now, OTIO adapter later for richer NLE interchange and metadata.

react render

Remotion

React compositions from typed tracks/items, Player preview, inputProps and deterministic render path.

html render

HyperFrames

plain HTML/CSS/media with timing data, seekable animations, CLI validation and FFmpeg output.

browser preview

WebCodecs / OpenCut

local decode/encode and editor/headless split for future visual reel dashboard.

automation

MCP / CLI tools

structured preflight, transcript, EDL build, render, verify and publish handoff.

quality gates

volumedetect / silencedetect

audio health, dead-air hints, first-frame checks, cut-boundary listening and final ffprobe report.

package docs

Markdown + JSON Schema

STACK.md, CONTRIBUTING, branch README, EDL skeletons and reusable config contracts.

cover lab

clean base + text layer

no-text cover bases, separate title/caption layer, 5-10 variants, contact sheet and small-size QA.

browser media

Mediabunny / WebCodecs

future local preview/export route with explicit containers, codec constraints and browser-native review.

monitoring

learning notes

first seconds, readability, audio pass, completion hints and survivor patterns written back into the skill.

минимальный технологический стек

must have
Python 3.10+, ffmpeg/ffprobe, Whisper or Scribe, Pillow, numpy, JSON EDL, sequential render queue, QA checklist.
macOS pack
Apple `avconvert` for iPhone HDR/HLG footage, then ffmpeg renders from SDR sources with bt709 tags.
caption pack
ASS/SRT/WebVTT for portable subtitles; Pillow/numpy for PNG overlays, contact sheets and review frames.
cover pack
clean no-text bases, independent text layer, variant contact sheet, identity/reference boundary and small-screen readability QA.
visual editor
browser board, tracks/items state, safe-zone overlays, contact sheets and review MP4.
adapter future
OpenTimelineIO export, Remotion or HyperFrames render adapters, Mediabunny/WebCodecs preview, MCP tools for agent control.
monitor loop
Every published or reviewed reel returns one learning note: keep, rerun, archive or promote mechanics into the public protocol.
